The file parsing system is responsible for transforming unstructured user uploads (CSV or XLSX) into a structured set of `CheckRow` objects. It implements a flexible column-mapping strategy to accommodate various naming conventions and enforces strict validation rules to ensure the DNS engine receives well-formed input.

## Implementation Overview

The logic is encapsulated in `fileParser.ts`, which utilizes `PapaParse` for CSV processing and `ExcelJS` for Excel files. The parser operates in three phases:

1. **Format Detection**: Identifying the file type by extension or MIME type.
2. **Column Resolution**: Mapping file headers to canonical fields (`hostname`, `type`, `value`).
3. **Row Validation**: Iterating through records to separate valid entries from `InvalidRow` entries.

## Column Alias Resolution

To support files generated by different tools or in different languages, the system uses a normalization and alias strategy.

### Canonical Mapping

The `COLUMN_ALIASES` constant defines the acceptable headers for each required field:

- **`hostname`**: `host`, `fqdn`, `name`, `nome`.
- **`type`**: `tipo`, `record`, `recordtype`.
- **`value`**: `valore`, `expectedvalue`, `atteso`.

### Normalization Logic

Headers are normalized using `normalizeHeader` which:

1. Trims whitespace.
2. Converts to lowercase.
3. Removes spaces, underscores, and hyphens using the regex `/[\s_-]/g`.

If any of the three canonical columns cannot be resolved from the file headers, the parser throws a `FileParseError`.

## Input Validation Rules

Each row is validated against a set of rules before being accepted into the batch. If a row fails any check, it is added to the `invalidRows` array with a concatenated string of error keys.

| Rule Key | Description | Implementation |
| --- | --- | --- |
| `emptyHostname` | The hostname field is missing or whitespace only. | `!hostname` |
| `invalidHostname` | The string does not conform to FQDN standards. | `isValidHostname(hostname)` |
| `emptyType` | The record type field is missing. | `!typeRaw` |
| `unsupportedType` | The type is not in the `SUPPORTED_RECORD_TYPES` list. | `isSupportedRecordType(typeRaw)` |
| `emptyValue` | The expected value field is missing. | `!value` |
| `mixedOperators` | The value contains both `&` and ` | ` (ambiguous logic). |

## File Format Details

### CSV Parsing

- **Library**: `PapaParse`.
- **Encoding**: UTF-8. The parser explicitly strips the Byte Order Mark (BOM) if present (`replace(/^﻿/, '')`).
- **Delimiter**: Auto-detected. Supports both commas (`,`) and semicolons (`;`).
- **Config**: Uses `skipEmptyLines: 'greedy'` to ignore rows that are technically present but contain no data.

### Excel (XLSX) Parsing

- **Library**: `ExcelJS`.
- **Sheet Selection**: Always processes the first worksheet (`worksheets[0]`).
- **Row Indexing**: Data extraction starts from row 2 (assuming row 1 is the header).
- **Empty Rows**: Rows where every cell is empty (after trimming) are silently skipped.

## Supported Record Types

The system supports standard DNS types and "policy" pseudo-types. During parsing, types are normalized via `normalizeRecordType` (uppercase and hyphen removal).

| Category | Types |
| --- | --- |
| **Standard** | `A`, `AAAA`, `CNAME`, `MX`, `TXT`, `NS`, `SRV`, `CAA` |
| **Policy** | `SPF`, `DKIM`, `DMARC`, `MTASTS`, `TLSRPT`, `BIMI` |
